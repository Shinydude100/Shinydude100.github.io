🛡️ Sentinel: [CRITICAL] Fix CSP inline script vulnerability

🎯 **What:** Removed the inline script block and inline `onload` attribute for font loading in `index.html`. The font loading logic has been refactored into `script.js`.

💡 **Why:** To comply with a strict `script-src 'self'` Content-Security-Policy (CSP) and remove the `unsafe-inline` directive, which is a significant vector for Cross-Site Scripting (XSS) attacks.

✅ **Verification:**
- Verified `index.html` structure with `npx htmlhint`.
- Confirmed no inline `<script>` tags or `onload` handlers remain.
- All Jest and Playwright tests pass successfully.

✨ **Result:** The application now strictly adheres to the defined CSP, improving the security posture against DOM-based XSS vulnerabilities.
