"Here is the exact specification for the WebMCP Declarative API. WebMCP tools are defined directly on HTML <form> elements using specific attributes, rather than using <link> tags.

Please locate the relevant forms in the HTML (such as contact forms, search bars, or interactive UI elements) and annotate them using this standard:

Form Attributes:
toolname: Required. The programmatic name of the tool (e.g., toolname="contactForm").
tooldescription: Required. A natural language description of what the form does so agents know when to use it.
toolautosubmit: Optional. Set to "true" if the agent is allowed to submit the form automatically without human confirmation.

Input Attributes:
toolparamtitle: A human-readable name for the field.
toolparamdescription: A description of what data the agent should inject into this specific field.

Example Implementation:
<form toolname="contactForm" tooldescription="Send a direct message or inquiry to Hugo Luna" toolautosubmit="true">
  <label for="email">Email</label>
  <input type="email" id="email" name="email" required toolparamtitle="Email" toolparamdescription="The user's email address" />

  <button type="submit">Submit</button>
</form>

Please apply these declarative WebMCP annotations to the relevant forms in the repository alongside the llms.txt file generation.
Carefully consider the request and update the plan with set_plan tool if needed."
